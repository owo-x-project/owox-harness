# 第1対象 CLI

## 状態

採用済み。

## 背景

実装順は「生成対象はまず Codex CLI」としていたが、選定理由が文書化されていなかった。
2026年6月に各 AI CLI の現行機能を調査し、判断を残す。

調査時点 (2026-06-11) の事実:

- Codex CLI は 2026年3月〜5月に hook (v0.124.0 安定化) ・skills (agentskills.io 標準準拠) ・subagents・rules・MCP resource 読取を出荷済み
- Claude Code は拡張性最充実だが、AGENTS.md をネイティブで読まない (CLAUDE.md からの import が公式回避策)
- Gemini CLI は hook・subagents・MCP resource 対応済み。AGENTS.md は設定追加が必要、既定化は却下済み

参照:

- `docs/decisions/20260611-制御方針.md`
- `docs/decisions/20260611-target-harness内容.md`

## 判断基準

- 制御方針の機械強制3点 (session 開始 / 編集・危険操作の前 / commit・stop 前) が成立する
- MCP resource の進行的開示が成立する
- ルート指示ファイル・skills・slash command の生成先がある
- 生成物が他 CLI へ移植しやすい (標準準拠)

## 採用案

第1対象は Codex CLI を継続採用する。

成立根拠:

- hook の PreToolUse が終了コード・JSON で決定的にブロックでき、機械強制の背骨が成立する
- rules (宣言的ポリシー、forbidden は決定的ブロック) を hook と併用でき、可逆性境界を二層で強制できる
- MCP resource 読取に対応し、owox:// の進行的開示が成立する
- skills がオープン標準 (agentskills.io) 準拠で、Claude Code とも可搬。マルチ CLI 展開の土台になる
- AGENTS.md をネイティブで読み、ルート指示ファイルの取り込み口が不要

## 制約と回避策

- commit 前の専用 hook イベントが無い → PreToolUse の Bash matcher で git commit を検査する。Stop イベントを完了検証に使う
- プロジェクト層の slash command が無い (公式却下済み、custom prompts 自体非推奨) → ショートカットの代替規定どおり skill のトリガ語で代替する (`docs/decisions/20260611-ショートカット.md`)
- hook 発火対象は Bash / apply_patch / MCP tool の3種のみ → owox の強制点 (危険操作・編集・commit) はこの3種で覆える
- 生成した hook は利用者が定義ハッシュ単位で承認する信頼モデル → setup が承認手順を案内する
- subagent にツール単位の許可リストが無い (sandbox_mode / mcp_servers 経由の制御のみ) → 第2段階のオーケストレーションで考慮。第1段階には影響しない

## 注意

- Codex CLI の hook は 2026年4月安定化と新しい。target repo 検証の最初期に hook 発火・ブロック・MCP resource 読取を確認する

## 捨てた案

- Claude Code を先にする
- Gemini CLI を先にする

## 捨てた理由

- Claude Code は AGENTS.md をネイティブで読まず、ルート指示ファイルの取り込み口生成が一段増える。拡張機構が独自拡張寄りで、最初に作ると生成物が Claude Code 固有に寄りやすい
- Gemini CLI も AGENTS.md に設定追加が必要。skills 対応の成熟度が未検証

## 見直し条件

- target repo 検証で Codex CLI の hook に成熟度問題が出た時
- 機械強制3点のいずれかが Codex CLI で成立しないと判明した時
- Codex CLI が AGENTS.md・skills 標準・MCP resource のいずれかの対応を後退させた時
- 第2段階のマルチ CLI 対応で第1対象の生成物が他 CLI へ移植しにくいと判明した時
