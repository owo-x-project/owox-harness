# ショートカット

## 状態

採用済み。

## 背景

人間の入口を整える。
skill はモデルが状況で自動起動、slash command は人間が明示的に叩く。補完関係。
コマンドは人間が owox フローを直接呼ぶ入口にする。

- `docs/decisions/20260611-方向付け.md`
- `docs/decisions/20260611-品質保証.md`
- `docs/decisions/20260611-タスク管理.md`

## 役割

- コマンド集合を commands.toml に正本定義し、各 CLI ネイティブの slash command 機構へ生成する
- コマンドは薄いトリガで、中身は MCP tool / skill を呼ぶ
- CLI に slash 機構が無い場合は skill のトリガ語で代替する

## Codex の実現 (Phase5 で転換)

一次情報で確認: Codex の custom prompts (slash の想定先) は廃止予定・ホーム限定 (`~/.codex/prompts/`) で repo 共有不可。公式が再利用指示は skills を推奨。
よって Codex では入口を skills へ寄せる。各コマンドを `.agents/skills/<command>/` の薄い skill として生成し、明示起動 ($名前) で MCP tool / resource を呼ぶ。誤った自動起動を避けるため入口系は allow_implicit_invocation=false。
詳細は `docs/decisions/20260613-Phase5-スキルと入口.md`。「slash 機構が無い CLI は skill で代替」の方針を Codex にも適用した形。

## コマンド

### 第1段階

- 立ち上げ・方向: /kickoff (立ち上げ) / /next (次の一手) / /status (現状)
- 判断・検証: /decide (判断支援) / /verify (検証・完了3区別) / /review (多視点レビュー)
- タスク: /task (ready・作成・クローズ) / /handoff (引き継ぎまとめ)

### 第2段階

- 生成・ゲート・経験: /generate (再生成) / /gate (不可逆操作の承認・ゲート解除。製品意図軸) / /export / /import

## 命名

- namespace 付きをデフォルトにする (例 /owox:next) 。衝突を避ける
- よく使うものは短い別名を設定できる

## 拡張

- owox 標準コマンド (固定コア) を持つ
- プロジェクトが commands.toml で独自コマンドを追加できる (skill / task へマップ)
- 拡張可能な正本にする

## 正本

- commands.toml: コマンド定義 (名前・別名・呼ぶ MCP tool / skill)

## 採用理由

- 正本定義して生成すると、CLI 非依存でコマンドが揃う
- 薄いトリガにすると、ロジックは MCP / skill 側に集まり重複しない
- namespace で衝突を避けつつ、別名で快適さも残せる
- プロジェクト定義可にすると、固有のよく使う操作をショートカットできる

## 捨てた案

- CLI ごとに手書き
- コマンドを出さず MCP 直叩きのみ
- 裸名のみ
- 固定コアのみ

## 捨てた理由

- 手書きは CLI 差でばらつく
- 直叩きのみは人間の入口が弱い
- 裸名のみは CLI 組込みや他ツールと衝突しうる
- 固定のみはプロジェクト固有操作をショートカットできない

## 見直し条件

- コマンドが増えすぎて選びにくくなった時
- 別名の衝突が起きた時
- slash 機構を持たない CLI を本格対応する時
- プロジェクト定義コマンドが乱立した時
