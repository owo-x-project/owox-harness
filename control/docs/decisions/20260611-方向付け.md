# 方向付け

## 状態

採用済み。

## 背景

成功条件の筆頭は「人間が次に決めることを迷わない」。
立ち上げ時に目指すものを定め、始めた後も方向を見失わない仕組みが要る。
grill-me (AI が人間を質問攻めにして計画の穴を炙り出す skill) の思想を採る。
ただし grill-me 最大の弱点は出力が揮発すること。owox は判断を残す道具なので、これを正本化で補う。

- `docs/decisions/20260611-製品戦略.md`
- `docs/decisions/20260611-設計原則.md`
- `docs/decisions/20260611-タスク管理.md`

## 立ち上げ: kickoff skill

- owox 標準 skill として出す
- grill-me 原則: 一度に1問 / コードで分かる事は自己解決
- 質問は3種で出し分ける（`docs/decisions/20260627-判断2軸と対話kickoff.md`）。引き出しは推奨なし・確認は構成提示・判断のみ推奨+理由+選択肢。「各問に推奨必須」は撤回（引き出しに推奨を貼ると押し付けになる）
- 引き出すのは製品意図軸（目的=Vision / 成功条件 / リスク許容 / 安全境界 / 優先順位）。行動軸（harness 構成・tool・skill 化）は owox が決めて確認に回す
- LIGHT に留める: 目的 (Vision) ・方針・初期タスク・初期判断を抽出する
- 出力を brand / 来歴 / tasks へ機械記録する (揮発させない)
- 深い要件仕様化はしない (owlspec の将来領域、対象外)
- skill ライフサイクルの dogfood になる

## 継続: 迷わない仕組み

### Vision (永続の目的)

- 単一の目的・価値を brand に常時持つ
- 全タスク・判断がそこへ trace する
- そこへ繋がらない作業 (orphan) を適応度関数で検知し、逸脱を表面化する

### 次の一手の提示

- session 開始で「次に決めること (open 未決) / 次にやるタスク (ready)」を出す
- 「次に人間が決める製品意図」と「AI が今すぐやる行動（行動軸）」を分けて出す。行動軸は owox が断定し、AI は人間へ打診も即興もしない（`docs/decisions/20260627-判断2軸と対話kickoff.md`）
- owox://next resource として出し、制御方針の session 開始で表面化する

### 未決の棚卸し

- open の来歴を一級で追跡・表面化し、解決を促す

### 判断支援スキル

- open 判断に、選択肢・トレードオフ・推奨を生成する
- 選択と却下を来歴に記録する
- owox 標準 skill

## 実装

- すべて skill + 既存プリミティブ (decision.record / task.create / brand / verify) で作る
- 新サブシステムは作らない

## 採用理由

- kickoff は迷わない入口を与え、出力の正本化で grill-me の揮発を解消する
- Vision は方向の drift を継続的に防ぐ
- 次の一手・未決棚卸しは、人間が次に決めることを常に明示する
- 判断支援を skill 化すると、選択肢比較と来歴記録が一体で回る
- skill + 既存プリミティブなので、対象外の汎用道具化を避けられる

## 捨てた案

- kickoff を入れない
- kickoff を要件仕様まで深掘りする
- 方向付けを専用サブシステムにする
- Vision を持たず drift を放置する

## 捨てた理由

- 入れないと目指すものを迷う入口が弱い
- 要件仕様まで深掘りは owlspec と重なり将来衝突する
- 専用サブシステムは汎用道具化に近づく
- Vision 無しは作業が目的から逸れても気づけない

## 見直し条件

- kickoff の質問が冗長で疲労を生んだ時
- Vision の trace 強制が作業を妨げた時
- 次の一手の提示が的外れになった時
- owlspec が要件仕様を扱い始め、kickoff と境界調整が要る時
