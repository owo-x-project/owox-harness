# 全体像

## 状態

俯瞰用。各決定の正本は個別文書。ここは地図。

## 一言で

検証可能な正本から target harness を生成・管理し、人間が重要判断を握ったまま AI の自律を最大化する。

## 流れ

立ち上げ → 正本 → 生成 → AI 作業 → 検証 → 記録 → 成長。

- 立ち上げ: kickoff skill が目的・方針・初期タスク・初期判断を抽出し正本化
- 正本: `.owox/` に価値・ルール・要件・品質・タスク・判断・経験を型付きで持つ
- 生成: 正本から各 AI CLI 向け target harness を生成 (型付き生成)
- AI 作業: 役割別 subagent が限定文脈で PEV (計画→実装→検証) を回す
- 検証: 完了3区別 + 品質ゲート + plan-alignment で機械検証
- 記録: 判断は来歴、経験はスキル、やることはタスクに残る
- 成長: 検証つきでルール・スキルが育つ。腐敗は機械検知

## 正本ソース .owox/

- brand.toml: 目的 (Vision) ・価値・方針・対象外・成功条件・用語集・表記文体
- rules.toml: 作業ルール・変更方針・依存追加条件・削除基準・可逆性境界・人間ゲート・安全性・トリガ
- quality.toml: 品質バー・依存方向・層境界・循環禁止・複雑度予算
- context.toml: 文脈地図 (作業 → 読むもの)
- state.toml: プロジェクト状態 (初期 / 安定 / 保守)
- agents.toml: subagent 役割・権限・モデルティア・オーケストレーションパターン
- commands.toml: 人間ショートカット
- permissions.toml: 役割・承認ポリシー (team)
- config.toml: machine 設定の集約。今は [targets.<cli>] (生成対象 CLI ・出力先・ティア → モデルマップ)
- release.toml: 配布方針・版・成果物検証 (必要な対象プロジェクトだけ)
- skills/: SKILL.md + tests + scripts + 経験メモリ
- requirements/: 要件・受け入れ基準・検証 link
- tasks/: 検証可能タスク (id / status / links / deps)
- decisions/: 来歴 (ID / 状態 / 根拠 / リンク)
- experience/: 汎用経験 (export 対象)
- verification/: 検証定義・結果
- work/: 作業メモ (git 管理対象外)

## 生成物 (各 CLI 規定位置)

- ルート指示ファイル / hook / MCP 設定 / slash command / skills 配置 / subagents / CODEOWNERS

## owox MCP

- resource (進行的開示): context / skills / brand / rules / quality / requirements / decisions / tasks / verification / state / next ほか
- tool (ドメイン別): harness / decision / requirement / task / verify / skill / experience / state / gate
- 共通返り値: status (ok / needs_human / failed) + reason + next_actions + 来歴 ID

## 制御の効き方

- 機械強制 (hook): 安全・不可逆・完了検証
- 指示 (最小): いつ何を使うか
- 誘導 (next_actions): 次の手
- 非準拠: 強制点は常にブロック、他は state 適応

## 決定文書の地図

- 製品戦略: アプローチ・旗 (検証可能な正本) ・差別化
- 技術スタック: Rust / 型構造体+TOML / GitHub Releases / 型付き生成 / MCP 主入口
- 開発アーキテクチャ: crate (core/mcp/setup) / Target / .owox/ / 経験分離
- 第1対象CLI: Codex CLI 継続採用・成立根拠・制約と回避策
- 設計原則: メモリ階層 / スキル・ライフサイクル / 可逆性境界 / 適応度関数 / 来歴 / 回帰防止 / ブランド固定
- target-harness内容: 正本ソース・生成物・MCP・hook・実装順
- MCP設計: resource と tool・共通返り値・人間ゲート・経験 IO
- 制御方針: 機械強制 / 指示 / 誘導・状態適応・トリガ
- タスク管理: 検証可能タスク (Beads 超え)
- 方向付け: kickoff・Vision・次の一手・判断支援
- 品質保証: plan-alignment・受け入れ基準 trace・品質適応度関数・回帰・多視点レビュー
- オーケストレーション: subagent パターン生成・役割権限・モデルティア・handoff
- ショートカット: 人間 slash command
- チーム権限: 役割ベース・CODEOWNERS 委譲・承認 identity
- eval方針: 検証つき成長の測定層・決定論優先・改善検証先・opt-in
