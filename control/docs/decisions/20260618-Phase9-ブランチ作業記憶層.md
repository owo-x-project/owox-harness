## 背景

git ブランチに紐づく作業記憶。フィーチャーブランチという作業単位ごとに「今このブランチで何をしているか・途中の判断・スクラッチ」を保持し、ブランチの生死に記憶の寿命を合わせる層。

既存記憶層との関係:

- decisions/ = 来歴 (恒久的判断)
- tasks/ = 検証可能タスク
- knowledge/ = 調査知識 (オンデマンド)
- experience/ = 汎用経験 (export 対象)
- practices = 成長層ルール (AI が育てる)
- work/ = 作業メモ (git 管理対象外・task.note・ただし global でブランチ非依存)

空白: 現状 work/ は global。ブランチをまたぐと混ざる/失われる。並行作業 (worktree・マルチエージェント) で文脈が衝突する。

## 採用設計

### 保管場所/git 管理 = work/ 拡張・ブランチ別キー・git-ignored 継続

新層を足さず、既に「作業メモ・git 管理外」の work/ をブランチ別キーにする。
scratch を git 履歴に載せない。worktree から同じブランチ記憶を引けるよう保管場所は1箇所に正規化する (working tree ごとに散らさない)。

### キー = ブランチ名・rename/rebase 自動追従なし

ブランチ名でキーを引く。rebase はブランチ名を変えないので問題なし。rename は稀なので自動追従は作らない。
孤児化したら腐敗検知に委ねる。ブランチ無し (detached HEAD) は global 退避。

### ライフサイクル = 自動削除しない・stale 検出で剪定・昇格は明示

merge/削除でいきなり消さない (確認していない物を消さない安全原則)。
ブランチが消えたら記憶は孤児 → 腐敗検知が stale として剪定候補に出す (庭師)。
恒久判断は decisions/・再利用経験は experience/ へ明示的に昇格 (自動でなく decision.record 等の行為)。
破棄基準 = ブランチ消失 + 経過日数。

### 並行作業の分離 = ブランチ別キーで自然分離

別ブランチの worktree は各々のブランチ記憶を見る。
同一ブランチのマルチエージェントは同じ記憶を共有する (意図通り)。
保管場所を正規化して worktree が同じバケツを引く。

### 性質軸との接点 = v1 は疎結合

delivery=phased + stage タグでブランチが stage に対応しがちだが、2つの新機構を絡めない。
ブランチ記憶が現在の stage タグを記録する程度に留め、強い機構は将来。

### マルチ CLI 共有 = v1 は同 repo・同ディスクのみ

同じ work/ をディスク共有するので同一マシンの複数 CLI は共有できる。
マシン跨ぎは git-track が要り履歴を汚すので対象外 (制限を明記)。

### 注意3点の扱い

- 最小コンテキスト: 床へ常時注入しない・オンデマンド読み (tool)。任意で「ブランチメモ N 件」程度の気づきのみ
- 秘密走査: 書込時に対象に含める (knowledge.add・experience と同じ入口の守り)
- 腐敗検知: stale-branch-memory として効かせる (放置ブランチ記憶を検出)

## 変更境界

触る:

- work/ をブランチ別キーへ拡張・保管場所の正規化 (worktree 横断で同じバケツ)
- ブランチ記憶の読み書き tool (オンデマンド読み)・書込時 secret 走査
- 腐敗検知に stale-branch-memory (孤児/放置の検出)
- 昇格導線 (decisions/・experience/ へ明示昇格)

触らない:

- decisions/・tasks/・knowledge/・experience/・practices の振る舞い
- 床コンテキストへの常時注入 (ブランチ記憶は非注入)
- 既存 work/ task.note の low-risk な作業メモ性質

## 危険

- worktree で保管場所が散る → 1箇所に正規化して同じバケツを引く
- ブランチ rename で記憶が孤児化 → 自動追従せず腐敗検知で stale 化し剪定候補に出す
- マシン跨ぎで共有できない誤解 → v1 は同 repo 同ディスクのみと制限を明記
- 放置ブランチ記憶が溜まる → stale-branch-memory + 庭師の剪定
- scratch に秘密が混じる → 書込時 secret 走査

## 検証方針

- 単体: ブランチ別キーの読み書き round-trip、ブランチ切替で記憶が分離、孤児の stale 判定、secret 検出で止める
- 結合: worktree/別ブランチで記憶が衝突しない、同一ブランチのマルチエージェントが共有、next/verify.run に stale-branch-memory が出る
- 手動 (実機): ブランチをまたいでも文脈が混ざらない、merge 後に記憶が剪定候補として出る、床へ常時注入されない (オンデマンド読み)

## 捨てた案

- 別層を新設 (.owox/branch-memory/ 等) → work/ が既に作業メモ・非git。新層を足さず拡張で足りる
- git-track してブランチと一緒に運ぶ → マシン跨ぎ共有できるが scratch が履歴/main を汚す (人間判断で却下)
- merge/削除時に自動破棄 → 未昇格の判断を失う。自動削除せず stale 化で剪定候補に
- ブランチ rename へ自動追従 → 稀な事象に機構を足すのは過剰。孤児化を腐敗検知に委ねる
- 床コンテキストへ常時注入 → 最小コンテキスト違反。オンデマンド読み

## 見直し条件

- マシン跨ぎ共有の実需要が出た時 (git-track 以外の共有手段の検討)
- 性質軸 (stage/delivery) とブランチ運用の強い連動が要ると分かった時
- stale-branch-memory の経過日数既定が実利用と合わない時
- worktree 横断の保管場所正規化が実機で破れる時
