# 任務種別ごとの行動軸切替

## 状態

採用・実装済み。`docs/requirements/20260625-mission-kickoff.md` の「各 tool は session 任務を読んで返却内容を変える」を、kickoff 以外の任務へも効かせる具体化。

## 背景

mission は work / kickoff / review / verify / handoff の 5 種。kickoff だけが実挙動を持ち
(専用質問構築・進行ブロック)、review / verify / handoff は context・next に一行の案内文を足すだけで、
利用可能 tool・next の手・verify 結果は work と同一だった。看板倒れ (`docs/validation` 系の所見)。

## 採用: mission は行動軸の焦点を切り替える

`docs/decisions/20260627-判断2軸と対話kickoff.md` が「ゲートは製品意図・安全・不可逆のみ」と定める。
よって mission で新しいゲートは作らない。mission が切り替えるのは **行動軸 (owox が断定する『今すぐ
やること』) と verify/context の焦点** だけ。これで原理 (ゲートの掛け先) を崩さず看板倒れを直す。

### next の mission 焦点

- review: 先頭行動を「差分を読む (context scope diff) → 所見を decision.record / task.create で記録。
  新規実装はしない」に。
- verify: 先頭行動を「verify.run → 未トレース要件を検査へ link → 作業範囲確認」に。
- handoff: 先頭行動を「検証状態 + 未決を要約 → 残作業を task 化 → handoff skill で引き継ぎ文」に。
- work: 焦点なし (既定)。kickoff: 専用の render_kickoff_next が担う。

非 Work mission は **保留が無くても沈黙しない**。空状態でも mission の作業を断定で出す
(看板倒れの主因は「空なら一律やることなし」だった)。

### verify.run の mission 焦点

検査結果自体は mission 不問で一貫させる (検査は正しい結果を返すべきで mission で歪めない)。
mission は next_actions の先頭 (どこへ向けるか) だけ切り替える。review=所見化、verify=要件完了の
中核、handoff=引き継ぎへ運ぶ。

## 捨てた案

- mission ごとにゲートを足す (review は編集を block 等) → 判断2軸の「ゲートは意図・安全・不可逆のみ」に反する。行動軸は断定で示すのが筋
- mission で verify の検査集合を変える (verify=フル / work=軽め) → 検査結果が mission 依存でブレ、混乱を生む
- 案内文一行のまま → 看板倒れが残る (今回の動機そのもの)

## 見直し条件

- review mission で新規実装を断定で止めたいだけでは足りず、機械ゲートが要ると分かった時
- 任務焦点が next の他セクションと衝突し読みにくくなった時
- 任務種別が増え、焦点定義の重複が無視できなくなった時
