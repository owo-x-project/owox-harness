# eval方針

## 状態

採用済み。

## 背景

未決「eval を本格導入する判断」を決める。
設計原則「機械検証を本筋、eval は段階導入」を具体化する。
eval は独立プロダクトでなく、検証つき成長の測定層。

- `docs/decisions/20260611-設計原則.md`
- `docs/decisions/20260611-品質保証.md`

## 位置づけ

- 決定論を優先する。配合の目安は 決定論60% (既存の verify / 適応度関数 / tests) + LLM-as-judge 30% + 人間10%
- eval は残差。決定論で測れない所だけ担う
- judge 単独で gate しない

## 主目的

改善検証を先、品質採点を後にする。

### 改善検証 (先)

- ルール / スキルの改善が本当に改善かを検証する
- golden set で新旧を paired 比較し、per-cohort delta で判定する
- 差分と結果を来歴に残す (完全版要件「改善前後の差分と検証結果」)

### 品質採点 (後)

- 決定論で測れない出力品質 (レビュー質・要件適合のニュアンス) を LLM-as-judge で採点する
- 段階的に追加する

## 手段

- 既存 eval ツール (DeepEval / Promptfoo 等) を orchestrate する
- golden-set / paired-delta の方法論と来歴連動は owox が持つ
- eval 道具自体は再発明しない

## 導入

- 既定 opt-in。ルール / スキル改善や評価が要る時に有効化する
- 既定 ON にせず、コストとノイズを抑える

## 健全性 (必須)

- golden set の正本化: .owox/ の verification 配下に versioned で持ち git 管理する
- paired + per-cohort delta: 新旧を同じ set で比較、cohort ごとの差分で判定。集計平均で局所退行を隠さない
- judge 校正: gate 前に LLM-judge を人間 gold で校正、position bias 対策 (順序入替)
- closed-loop: 失敗トレースを golden set へ還流し、段々騙せなくする
- 決定論優先: eval は決定論の残差。judge 単独で gate しない

## 正本

- verification/ 配下に eval 定義・golden set・judge 校正を置く

## 採用理由

- 決定論優先は、judge の stochastic を ground truth にしない
- 改善検証を先にすると、完全版要件の改善差分要求に直接応える
- paired + per-cohort delta は、集計平均が隠す局所退行を捕まえる
- judge 校正と順序入替は、judge のバイアスを抑える
- closed-loop は、eval を成長させ腐敗を防ぐ
- orchestrate は、eval エコシステムの恩恵を受けつつ再発明を避ける
- opt-in は、初期のコストとノイズを抑える

## 捨てた案

- LLM-judge 単独で gate
- eval を owox が独自フル実装
- 既定 ON で全 PR
- 集計平均だけで判定
- golden set を正本化しない

## 捨てた理由

- judge 単独は scorer の stochastic が乗り信頼できない
- 独自フル実装は再発明で重い
- 既定 ON は初期で重く詰まりやすい
- 集計平均は局所退行を隠す
- 正本化しない golden set は腐り、再現できない

## 見直し条件

- 改善検証で golden set 運用が重くなった時
- judge 校正のずれが大きくなった時
- 品質採点を本格導入する判断がついた時
- eval を既定 ON にする判断がついた時
- orchestrate するツールの選定を見直す時
