## 見つけた粗

### 粗1: 調査記録が practices へ誤誘導 (発見性)

「調査結果を残したい」に対し AI が `knowledge.add` でなく `canon.add target=practices` を選んだ。原因は SessionStart 床コンテキストの「Acting on what the human asks」routing 一覧に knowledge が無く、最も近い「lasting practice to follow」へ寄せたため。get_info の MCP instructions には記載済だったが、routing を駆動するのは床コンテキストの一覧であり効かなかった。

### 粗2: 頻出手順の提案がノイズ

`next` の routine 節の最頻提案が「Bash → Bash → Bash → Bash」だった。使用履歴は name のみ記録 (引数は残さない) のため、シェル操作の連打が同一 name の長い連続になり支配的になる。技にできる「手順」でない。

## 是正と再確認

- 粗1: `floor_context` の routing へ knowledge 行を追加 — 「調査結果は knowledge.add、recall は knowledge.lookup。これは research knowledge で、従う指針 (canon.add) とは別」と明記し practices と区別。再起動後、床に knowledge 行が出て AI が `knowledge.add` を正しく呼ぶことを確認
- 粗2: `run_routine_suggestions` に distinct な name 2種以上の要求を追加 (同一 name 連打は手順でないので除外)。再確認で routine 節から「Bash 連打」が消え、現 usage では資格を持つ手順が無く節自体が出ない (ノイズを正直に抑制)。distinct な多段手順は単体テストで表面化を担保
