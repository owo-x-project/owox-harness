## 改善考察

人間所見「kickoff が機械的で grill-me のような共同決定がない。AI が考えを押し付ける。kickoff 外でも AI が雑に人間へ判断を仰ぐ」を起点に分析。
理想「行動の判断は全て owox が示す。人間は何をどう作りたいかだけ示す。AI が必要に応じて助言し実作業する」。

### 核心: 判断を 2 軸に分けていない

理念正本（`docs/concept/企画書.md`）の「人間が重要判断を握る」と上記理想を合わせると、判断は 2 種類。

- 製品意図軸（人間が握る）: 何を / 誰のため / 成功条件 / リスク許容 / 止める操作 / 優先順位
- 行動軸（owox が握る）: どの tool をいつ使うか / verify 実行可否 / skill 化 / profile / rules 下地 / 索引更新 / 庭師

今の kickoff と tool 群はこの 2 軸を混同。結果:

- 行動軸まで人間へ質問 → 「雑に判断を仰ぐ」
- 製品意図軸を引き出さず即推奨提示 → 「考えを押し付ける」

理想はこの 2 軸の分離そのもの。「行動判断は owox が示す」= 行動軸を owox 所有。「人間は何をどう作るかだけ示す」= 製品意図軸を人間から引き出す。

### kickoff が機械的な理由（`docs/requirements/20260625-mission-kickoff.md` 構造由来）

- 固定 10 段階チェックリストを `next` が 1 件ずつ行進 → フォーム記入感（自販機型）
- 「全質問に推奨案+理由を必須」(要件 127-128 行) → 引き出すべき開いた質問にも推奨を貼る → 押し付け感
- 段階 6/7/8（harness 構成: rules / skill / tool）は本来 owox が決める行動軸なのに人間質問化
- 人間が tool を直接叩く前提 → 対話でなく操作

grill-me が要るのは製品意図軸だけ。そこは推奨提示でなく「引き出し → 理解を反映 → 鋭い追撃 1 問 → 共同で収束」の対話。今は全部「決めること:X / 推奨:Y / 決定者:human」の自販機型。

### 検証所見の再解釈

- P1 `.owox/.gitignore` 自動生成 → 新モデルでは行動軸=owox 所有で「書いてよい」。矛盾は「全書込を人間 gate」にした要件側の誤り。gate は製品意図正本（requirements / decisions）だけに掛ける
- entry-routing 常時 advisory / `tool_search` 指示外 / 余分な `verify.run` → 全部「行動方針が権威的でない」症状。owox が次の行動を断定指示しないから AI が即興 or 打診する
- `next` が即「人間判断待ち」→ 引き出し対話ゼロの証拠

### 改善案

- 判断 2 軸を正本化（`docs/decisions`）。全 tool・skill はこの軸で振る舞いを決める土台
- kickoff を対話型へ。質問を 3 種に分け `next` が種別を返す
  - 引き出し: 開いた質問。推奨なし可。AI は理解を反映し鋭い追撃 1 問
  - 確認: AI が決めた構成・理解を提示。人間は否認 / 修正だけ
  - 判断: 人間しか決められない意図 / 安全の分岐のみ。推奨+理由+選択肢
  - 「全質問に推奨必須」を撤回し種別で出し分け。段階 6/7/8 は質問でなく owox 決定 → 確認へ格下げ。人間は自然発話、AI が owox を代理操作
- 行動軸の権威化（kickoff 外の「雑に仰ぐ」対策）。`next` は「次に人間が決める意図」と「AI が今すぐやる行動」を分けて返す。行動軸は owox が断定。AI は打診も即興もしない。人間 gate は意図正本・安全境界・不可逆操作のみ
