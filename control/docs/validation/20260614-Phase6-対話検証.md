## 見つけた粗と是正 (crates/mcp/src/files.rs)

### 粗1 quality 走査が未追跡ファイルを見ない

- list_repo_files が git ls-files (tracked のみ) で、新規未コミットのソースの quality 違反を verify.run / commit ゲートが検出できない (git add 後は正常)
- 是正: git_untracked ヘルパ (ls-files --others --exclude-standard) を抽出し list_repo_files に合流。.gitignore 除外を維持しつつ新規ソースも走査対象へ

### 粗2 changed_files の偽カウント

- git diff --name-only -z HEAD が空変更で改行のみを出力し、git_lines の filter(!is_empty) が除外できず偽の変更ファイルになる
- 是正: filter を trim_ascii ベースへ (!s.trim_ascii().is_empty())。git 挙動に依存せず NUL 区切りのゴミを除去

### 是正の確認

- 単体: cargo test 146 passed (140→146)、clippy clean、fmt clean
- 結合: target に未追跡の違反ファイルを置き verify.run の data.quality に budget・boundary を検出、changed_files が実数 (3) へ。後片付け済み
- 触ったのは files.rs のみ、core / serve は不変
