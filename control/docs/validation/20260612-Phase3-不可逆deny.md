## 生成した target harness

`owox generate codex /workspace/product/target` で 2 件配置:

- AGENTS.md: ルート指示ファイル
- .codex/hooks.json: SessionStart (startup) と PreToolUse (Bash) を、command で owox を直接呼ぶ形に登録

変更点 (Phase 1 から):

- 薄いシェル `.codex/hooks/session-start` を廃止 (実行ビット付きファイルも撤去)。command を `owox hook session-start` / `owox hook pre-tool-use` の直接呼び出しに
- PreToolUse (matcher Bash) を追加。owox が不可逆操作を deny する

owox: release ビルドを `/usr/local/bin/owox` へ再配置 (PATH 上)。

## 実行した確認 (機械)

`owox hook pre-tool-use` に Codex の PreToolUse 入力 JSON を stdin で渡し、出力を確認:

- deny: `git push --force` / `rm -rf build` / `git reset --hard HEAD~3` で、`hookSpecificOutput.permissionDecision=deny` + `permissionDecisionReason` を stdout へ返し終了コード 0。出力は一次情報の deny 契約と完全一致
- allow: `git status` / 非 Bash tool (apply_patch) で出力無し・終了コード 0 (素通り契約)
- 既定検出器の単体テスト: force push / reset --hard / rm 再帰強制 / git clean -f / git branch -D を検出。通常 push・force-with-lease・単発 rm・`rm -r`・`git clean -n` は素通り (誤発火回避)
- 再生成の冪等性・hooks.json スキーマ (hooks ラッパ + キャメルケースのイベント名) は既存テストで維持
- cargo test 18 本通過 / clippy 警告なし / fmt 済み

## 結果 (機械)

不可逆操作の検出 → deny 出力の配線が成立。出力契約は一次情報と一致。

## 確認手順 (人間、TTY のある実ターミナルで)

`codex exec` は未承認 hook をスキップするため検証不可 (Phase 1 と同じ制約)。実ターミナルで対話起動する。

1. `cd /workspace/product/target`
2. `codex` を対話起動
3. hook 定義の信頼承認を求められたら**承認** (定義が変わったため Phase 1 で承認済みでも再承認が要る)
4. SessionStart 確認: 開始直後に「owox の文脈で『次に読むもの』は何か。ファイルは読まず与えられた文脈だけで答えよ」→ owox://context / owox://next を答えれば注入維持
5. deny 確認: AI に安全な不可逆操作を試させる。例「`git reset --hard HEAD` を実行して」→ PreToolUse で deny され、理由 (Confirm with a human first) が出れば成立
6. 誤発火確認: 通常操作 (`git status` / `ls` / `git push`(force なし)) が妨げられないこと

注意: 手順 5 は実際に破壊が起きないコマンドで試す (deny されれば実行されないが、承認を誤って通すと実害が出るものは避ける)。
