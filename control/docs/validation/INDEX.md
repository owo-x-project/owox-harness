# validation

`target repo` での検証結果を置く。

## 置くもの

- 検証した日時
- 検証した target repo
- 生成した target harness
- 実行した確認
- 結果
- 未確認
- 次に確認すること

## 読む時

- 実装が期待通りか確認する時
- 失敗の再現条件を見る時
- 次の検証範囲を決める時
