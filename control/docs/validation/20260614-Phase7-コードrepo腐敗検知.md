# Phase7 スライス2 コード/repo 腐敗検知の検証

## 検証対象

Phase7 スライス2 (`docs/decisions/20260614-Phase7-コードrepo腐敗検知.md`)。
庭師の検出側をコード/repo へ拡大する。owox 直接検出は内容一致の重複ファイル、死コード等は委譲。

- core: decay.rs (run_code_decay = 重複ファイル検出 + 委譲検査 run_decay_checks)、quality.rs (DecayConfig へ min_duplicate_bytes・checks、`[[decay.checks]]` 解釈)、lib.rs (公開)
- mcp: serve.rs (verify.run が run_decay と run_code_decay を結合)

検証する owox の版: 本作業ツリーの release ビルドを `/usr/local/bin/owox` へインストール。target サンドボックス (実 target の一時コピー) で MCP stdio/hook 確証。

## 結果

### 単体 (cargo)

- cargo test 174 passed (171→174。重複ファイルの検出/極小無視・decay-check の非ゼロ終了報告を追加)
- clippy clean・fmt clean

### MCP stdio (サンドボックス: 重複ファイル2つ + [[decay.checks]] に非ゼロ終了コマンド)

- verify.run: data.decay に `duplicate-file` ×2 (dup_a.rs ↔ dup_b.rs・相互に相手パスを detail へ) + `decay-check` (dead code probe・exit 3 を要約) を報告
- next: repo decay (duplicate-file・decay-check) は出ない。タスク・来歴 decay の高速パスを維持 (設計どおり)
- 重複検出は内容一致で誤検出なし。min_duplicate_bytes 未満の極小ファイルは対象外

### commit ゲート (hook): コード decay は advisory

- 保守 (maintenance) phase + コード decay のみ (構造的タスク腐敗なし) → commit は ALLOW (止めない)
- コード decay (重複ファイル・decay.checks) は advisory で commit を止めない。止めるのは引き続き構造的タスク腐敗 (done未検証・ゾンビ) のみ

### 線引きの確認

owox 直接検出は内容一致の重複ファイルだけ (言語非依存・決定論)。死コード・未使用は `[[decay.checks]]` でプロジェクトの検査コマンドへ委譲し、owox が走らせ非ゼロ終了を decay として報告する (advisory)。完了を判定する verify.checks (pass/fail ゲート) と区別。報告は verify.run の data.decay に集約し、next は高速・commit は止めない。
