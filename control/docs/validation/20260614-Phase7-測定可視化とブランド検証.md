## 結果

### 単体 (cargo)

- cargo test 191 passed (174→191。トークン推定・秘密走査・Glossary Forbidden 解釈・run_brand・glossary.add 保護を追加)
- clippy clean・fmt clean

### MCP stdio (サンドボックス: glossary.md に禁止語・追跡文書に禁止語・context.md に秘密)

- context tool: 描画末尾に `Estimated size: ~133 tokens.` を表示 (owox が注入する文脈量を数値化)。context.md に仕込んだ AWS アクセスキー形式の文字列を `Secret warning [aws-access-key-id]` として警告 (秘密の値そのものは警告に載せない)。read 専用ナビなので block せず描画は継続
- verify.run: data.quality に `{kind:"brand", path:"note.md", detail:"contains forbidden term /\\bfoobar\\b/ (造語禁止。代替を使う)"}` を報告。quality 違反と同じチャネルに合流し完了3区別は変えない

### commit ゲート (hook): ブランド違反の phase 適応

- 初期 (initial, 既定) phase: brand 違反は additionalContext の警告のみ・exit 0 (ALLOW)。「Address them, or relax the rule in quality.toml.」
- 保守 (maintenance) phase: brand 違反で permissionDecision=deny。「1 quality bar violation(s) and the project is in maintenance ... Fix them, or relax the rule ... before committing.」
- quality バー違反 (boundary/budget/brand) を kind 不問で扱う既存 commit_gate がそのまま適応。新ゲートを足していない

### 線引きの確認

- トークン量は日本語対応の軽い推定 (CJK 1 文字 ≒ 1 トークン・他 4 文字 ≒ 1 トークン)。正確なトークナイザは入れず新規依存なし。桁感を示す表示で pass/fail ゲートにしない
- 秘密走査は irreversible 方式の再利用 (組込み正規表現)。文脈出力は警告のみで block しない。export/import (スライス4) が本命の再利用先
- 禁止語は glossary.md の予約見出し Forbidden に正規表現で構造化 (用語集が用語+定義+禁止語の正本)。正規表現で表せる表記規則は既存 quality.toml boundaries を流用 (新機構なし)。助詞省略など構文解析が要る文体は対象外
