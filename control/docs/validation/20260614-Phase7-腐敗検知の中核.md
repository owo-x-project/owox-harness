### commit ゲート (hook) の phase 適応

- 保守 (maintenance): 構造的腐敗 (done未検証 + ゾンビ) を deny。理由に2件を列挙
- 初期 (initial): deny せず remind のみ (open 来歴 + 構造的腐敗を additionalContext で列挙)
- age 系 (放置/孤立/重複/来歴鮮度) は保守でも commit を止めない (構造的腐敗だけがゲートへ届く設計どおり)。狼少年化を回避

### 線引きの確認

腐敗検知は quality と同じ「機械検出して報告、効かせ方は commit で phase 適応」の型に載せた。閾値は quality.toml の `[decay]` (既定で検知)。古さは ID 日付と note 日付の最新と today の差 (core は git/時計を持たず today を mcp が供給)。commit を止めるのは構造的腐敗のみ、age 系は next/verify.run の助言に留める。

## 失敗と再現手順

### 検証中に発見し是正した粗 (1件)

- 現象: dropped タスクが orphan (孤立) 誤検出された。再現: status=dropped・link/dep 無し・古い日付のタスクを置き next/verify.run を呼ぶ
- 原因: orphan が status を見ておらず終了済みタスクも対象にしていた (放置は active 集合に限るのに孤立は限っていなかった)
- 是正: orphan を放置と同じ active 集合 (todo/doing/blocked) に限定 (decay.rs)。回帰テスト `terminal_tasks_not_orphan` 追加
- 確認: 是正後の再走で abandoned-prereq (dropped) が無印になり、検出が 12→11 件・orphan 4→3 件へ
