## 作るもの

### MCP コンテキスト削減
- 生成後除去 (B 方式・owox 完結): inputSchema から title・null/空配列 default を全 route で除く。`crates/mcp/src/serve.rs` clean_schemas
- inputSchema 冗長除去: フィールド説明の括弧補足を本文へ畳む
- ツール本体 description 短縮: 意味と発見性を保ち重複冗長のみ削る
- `$schema` は実機 Codex 受理を確認するまで残す (安全弁)

### 自動承認パス再設計 (旧点2 を supersede)
- 自動承認の既定を architecture 軸から導く純関数: flat=オン・layered=オフ。`crates/core/src/profile.rs` auto_approval_default
- 同意源2系統: profile 由来=永続・session 由来=session 限り。実効 auto はどちらかが立てば有効。`crates/mcp/src/serve.rs` AutoApproval
- 対象集合は層 (supervised/free)×不可逆 (除外) の積で機械決定。許可リストを持たない
- 不可逆操作は層・auto と直交する独立最優先ガードが常に止める (穴A)。既存 hook pre_tool_use を再利用
- 後追いキューは来歴一本 (approval:auto 印+未確認)。next で未確認を表面化・gate.confirm/gate.revert で確認差戻し

### オーケストレーション
- 正本 agents.toml (CLI 非依存) 新設: 役割5 (調査・計画・実装・レビュー・検証)×変種+責務+ティア+sandbox+許可層+文脈。組込み既定 + 部分上書き。`crates/core/src/agents.rs`。`.owox/agents.toml` で上書き
- ティア写像: config.toml [targets.<cli>.models] に tier→(model, reasoning_effort)。`crates/core/src/model.rs` ModelTier
- 床の能動提示: 床コンテキストに `## Orchestration` 節 (骨格・役割・変種・スケール適応・並列・handoff)。`crates/core/src/hook.rs`
- Codex 写像: 実機で custom agent_type spawn 不可と判明 (`docs/validation/20260620-Phase9-オーケストレーション実機.md`)。設計の安全弁どおりフォールバック (組込み agent_type + spawn 時 prompt/developer_instructions 注入・床が駆動) を採用。`.codex/agents/*.toml` 生成は撤去

## 作らないもの

- 自動承認の明示許可リスト (層×不可逆の積で機械決定)
- 人間と同等権限の新層・autonomy 5軸目 (既存自律度勾配の閾値調整で済ます)
- 後追いキューの専用ファイル/cache (来歴一本)
- 役割×変種ごとの TOML (変種は spawn 時 prompt 上書き)
- 具体モデルの正本直指定 (ティア4段で吸収)

## 完了条件

- MCP: tools/list で title・null/空配列 default が消え、$schema は保持。description が短い
- 自動承認: flat で auto 有効・layered で無効。実効 auto が profile/session の OR で決まる。不可逆は迂回されない
- オーケストレーション: agents.toml が役割×変種で解決され、床が役割と呼び出しルールを能動提示する
- cargo 通過・clippy 0。実機 Codex で cleaned schema 受理・床注入を確認
