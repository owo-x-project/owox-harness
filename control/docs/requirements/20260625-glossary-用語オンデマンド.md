# 用語オンデマンド

## 状態

草案。次回以降に詳細を詰める。

## 背景

用語定義を常時注入すると文脈が重くなる。
一方で用語名を知らないと `glossary.lookup` に到達しにくい。
発話・読取対象・明示 lookup の3経路で、必要な時だけ用語定義を届けたい。

tool は増やさず、既存の hook と `glossary.lookup` を強める。

## 目的

- 用語定義を完全オンデマンドに近づける
- 用語名一覧の床注入を減らす
- ファイル読取前に対象内容から用語を検出して定義を届ける
- 用語候補の取得も `glossary.lookup` へ寄せる

## 対象

- `SessionStart`
- `UserPromptSubmit`
- `PreToolUse`
- `glossary.lookup`

## 対象外

- 新規 glossary 専用 tool
- 用語定義の常時全量注入
- `.owox/` の直読み許可
- 生ファイル本文の常時解析保存

## 仕様

用語配送を3段にする。

1. `SessionStart`
   - 用語一覧は原則出さない
   - 出すのは案内1行だけ
   - 定義は出現時に届く、必要なら `glossary.lookup` を使うと伝える

2. `UserPromptSubmit`
   - ユーザー発話を走査する
   - 出た用語だけ定義を注入する
   - 既に同 session で注入済みの用語は再注入しない

3. `PreToolUse`
   - 読取前に対象ファイルを軽く読む
   - 出た用語だけ定義を注入する
   - その後、通常の読取を通す

## 読取対象

初期対象:

- `Read`
- `cat`
- `sed -n`
- `head`
- `tail`

後回し:

- `rg`
- `grep`

理由: 複数ファイル・標準入力・巨大出力が絡みやすい。

## 安全線

- リポジトリ内だけ読む
- バイナリを読まない
- 最大読み取り量を決める
- `.owox/` 直読みは既存方針どおり止める
- session 単位で重複注入しない
- 読めない時は作業を止めず素通りする

## `glossary.lookup`

新しい tool は作らず、`glossary.lookup` を広げる。

- 完全一致: 定義を返す
- 別名一致: 正規用語と定義を返す
- 部分一致: 候補を返す

候補取得により、用語自動サジェストも同じ読み口で扱える。

## 用語登録サジェスト

新しい tool は作らない。
`verify.run` / `next` / `context scope="diff"` に候補を出す。

候補は、AI が知らない単語ではなく、プロジェクト固有っぽく何度も判断に効きそうな語に絞る。

検出元:

- 差分内の新しい文書
- 差分内のコメント
- 要件・来歴・タスク本文
- ユーザー発話
- `glossary.lookup` で見つからなかった検索語
- 読取前スキャンで何度も出る未登録語

候補条件:

- 同じ語が複数ファイルに出る
- `PascalCase` / `snake_case` / `kebab-case` / 英字略語 / 日本語の固有っぽい連続語
- 複合語
- 要件・来歴の表題に出る
- ユーザーが何度も使う
- lookup 失敗後に同じ語が再出現する

除外:

- 一般語
- 予約語・言語構文
- ファイル名だけの語
- 既存用語・別名
- 短すぎる語
- 数字だけ

出し方:

- `verify.run`: `data.glossary_suggestions` に助言として出す。止めない
- `next`: 用語候補ありだけ出す。多すぎる場合は上位だけ
- `context scope="diff"`: 差分に出た候補と根拠ファイルを返す

登録は既存の `canon.add` を使う。

```text
canon.add section="glossary" text="用語: 定義"
```

AI は候補を出す。定義は勝手に確定しない。
定義案を出す場合も案として人間確認へ寄せる。

候補には根拠を必ず付ける。

```text
term: target harness
reason: 3 files, requirement title, user prompt
examples:
- docs/requirements/...
- crates/...
```

根拠なしサジェストは禁止。

## 検証

- 発話に出た用語だけ定義が注入される
- `Read` 対象ファイル内の用語だけ定義が注入される
- `cat` / `sed -n` / `head` / `tail` の対象ファイル内の用語だけ定義が注入される
- 同 session で同じ用語を再注入しない
- `.owox/` 直読みは許可しない
- 大きいファイルで最大読み取り量を超えても止まらない
- `glossary.lookup` が完全一致・別名一致・部分一致候補を返せる
- `verify.run` / `next` / `context scope="diff"` に根拠つき用語候補を出せる
