# 伝言メモ不要

## 状態

草案。次回以降に詳細を詰める。

## 背景

プロジェクト固有の伝言メモ的な機能が必要か検討した。
既に用途別の記録層があり、新しい伝言メモ tool を足すと保存先が増えて迷う。

## 判断

新機能としての伝言メモは作らない。
代わりに、既存の記録先へ分類する導線を強める。

## 既存の保存先

- durable な判断: `decision.record`
- タスク内の一時メモ: `task.note`
- ブランチ作業メモ: `branch.note` / `branch.notes`
- 調査結果: `knowledge.add`
- スキル改善メモ: `skill.remember`
- 次セッション引き継ぎ: `$handoff`

## 仕様

ユーザーが「メモして」「次の人に伝えて」と言った時、AI は内容で保存先を選ぶ。

- 後で覆してはいけない判断 → `decision.record`
- 今のタスクの作業状態 → `task.note`
- このブランチ限定の作業状態 → `branch.note`
- 調査で分かった事実 → `knowledge.add`
- スキルの改善点 → `skill.remember`
- 次セッション全体への引き継ぎ → `$handoff`

## 実現方針

新規 tool は作らない。

- `$handoff` を強化する
  - `next`
  - `verify.run`
  - `context scope="diff"`
  - open gate
  - ready task
  - branch notes
  - stale items

- `$task` / `$decide` / `$skill` にメモ分類ルールを足す
- `SessionStart` には詳しい分類を書かない

## 対象外

- 伝言メモ専用 tool
- 新しい永続ストア
- 会話ログ依存の引き継ぎ
- 分類不能なメモの自動保存

## 検証

- 「メモして」依頼が既存の適切な保存先へ流れる
- `decision.record` と `task.note` の使い分けができる
- ブランチ限定の内容が `branch.note` へ流れる
- 調査結果が `knowledge.add` へ流れる
- 新規 tool が増えない

## 未決事項

- `$handoff` の最終手順
- 分類不能な時に人間へ確認する文言
- `$task` / `$decide` / `$skill` の本文変更範囲
