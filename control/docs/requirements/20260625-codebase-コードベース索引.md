# コードベース索引

## 状態

草案。次回以降に詳細を詰める。

## 背景

コードベースを調査した汎用情報を毎回再調査すると無駄が多い。
ただし、コードから再生成できる情報を永続正本にすると腐る。
コードベース調査結果は記憶ではなく、読む場所を探すための索引 cache として扱う。

## 目的

- AI が読むべき場所を探しやすくする
- コードベースの構造地図を再利用する
- 古い情報を信じない
- 常時注入せず、必要時だけ使う

## 対象

- `context scope="codebase"`
- `context scope="diff"`
- `verify.run`
- `next`

## 対象外

- ソース本文コピー
- AI の長い説明
- 古いコードの要約
- 推測の保存
- git 管理する恒久記憶
- experience export
- `context scope="manifest"`

## 仕様

名前は `codebase index`。日本語ではコードベース索引。

専用 manifest は持たない。
読む場所の地図は `context scope="codebase"` に寄せる。

保存する情報:

- ディレクトリの役割
- 主要 entrypoint
- 重要な設定ファイル
- test / build / lint の入口
- 層っぽい構造
- 依存関係の大枠
- generated / vendor / ignored の候補
- よく参照されるファイル
- 最近調べた関連ファイルセット

保存しない情報:

- AI の長い説明
- 古いコードの要約
- 推測
- ソース本文コピー
- 壊れやすい振る舞い説明

## 呼び出し導線

`codebase` は、AI が読む場所に迷った時に呼ぶ地図。

入口 skill:

- `$next`
  - 次に触る範囲が曖昧なら `context scope="codebase"`
- `$review`
  - 差分確認前に `context scope="diff"`
  - 構造理解が必要なら `context scope="codebase"`
- `$task`
  - タスク着手前に関連領域が不明なら `context scope="codebase"`
- `$kickoff`
  - 厚い導入として `context scope="codebase"` を使う

`SessionStart`:

```text
Use context scope="codebase" when you need a repo map before choosing files to read.
```

`context scope="diff"`:

- 変更ファイル分類が不明なら `context scope="codebase"` を next action に出す

## 返す情報

本文や生コードは返さない。
読むべき場所を選ぶための地図だけ返す。

```text
scope: codebase
root:
  kind: rust-workspace
  package_files:
    - Cargo.toml

areas:
  - path: crates/core/src
    kind: source
    role: core logic
    evidence:
      - crates/core/src/lib.rs
      - crates/core/src/model.rs

  - path: crates/mcp/src
    kind: source
    role: CLI and MCP surface
    evidence:
      - crates/mcp/src/serve.rs
      - crates/mcp/src/hook.rs

entrypoints:
  - crates/mcp/src/main.rs
  - crates/mcp/src/serve.rs

checks:
  - cargo test
  - cargo clippy

generated_or_external:
  - target/
  - .agents/

related_to_current_diff:
  - docs/requirements/...
  - crates/core/src/hook.rs

stale:
  false
```

`role` は断定しすぎない。根拠ファイルから機械的に言える範囲だけにする。

## 更新と腐敗検知

索引には根拠を持たせる。

```text
source_files:
- Cargo.toml
- crates/core/src/lib.rs
git_head: <sha>
generated_on: 20260625
```

腐敗条件:

- 根拠ファイルが変わった
- 根拠ファイルが消えた
- git_head が古い
- 参照先が存在しない
- 一定日数経過

腐ったら捨てるか再計算する。人間ゲートは不要。

## 昇格

索引は memory ではなく cache。

- 調査して分かった耐久事実 → `knowledge.add`
- 設計判断 → `decision.record`
- 作業途中 → `branch.note`

## 検証

- `context scope="codebase"` が本文なしの地図を返す
- `context scope="diff"` が必要時に codebase を促す
- stale な索引を検出できる
- 腐った索引を信頼しない
- 常時注入しない
- experience export に含めない
