# 参照IDと参照検査

## 状態

草案。次回以降に詳細を詰める。

## 背景

ソースコードや文書から、どの要件・判断に基づく変更か追えるようにしたい。
ただし MCP tool は増やしすぎない。参照専用の細かい tool 群は作らず、既存の検証と文脈取得へ寄せる。

## 目的

- ソースコード・文書から要件と判断を参照できる
- 壊れた参照を機械検出できる
- 参照先から利用箇所を逆引きできる
- 正本差分レビュー・庭師機能・腐敗検知の根になる

## 対象

- `.owox/requirements/<id>.md`
- `.owox/decisions/<id>.md`
- ソースコード内コメント
- docs 配下の文書

## 対象外

- 新しい短縮ID台帳
- 参照専用 tool の分割追加
- 参照の自動挿入
- 要件・来歴ID体系の作り直し

## 仕様

参照記法を固定する。

```text
owox:req:<requirement-id>
owox:req:<requirement-id>#<criterion-id>
owox:dec:<decision-id>
```

- `owox:req:<requirement-id>`: 要件全体への参照
- `owox:req:<requirement-id>#<criterion-id>`: 要件内の受け入れ基準への参照
- `owox:dec:<decision-id>`: 判断・採用理由への参照

既存の要件ID・来歴IDをそのまま使う。短縮IDが必要になった場合だけ、要件・来歴側へ別名を持つ案を再検討する。

## 実現方針

新規 tool は原則作らない。

- `verify.run`
  - 参照記法を走査する
  - 参照先が存在するか確認する
  - 要件内の受け入れ基準番号が存在するか確認する
  - 壊れた参照を `data.references` に出す

- `context`
  - `reference_id` 引数で参照IDの逆引きを返す
  - 近い要件・来歴の候補を返す
  - 参照先本文の読み口を既存の文脈取得へ寄せる

- `requirement.get`
  - 要件本文と受け入れ基準を読む

- `decision.record`
  - 既存の来歴記録を維持する
  - 来歴の読み口が不足する場合だけ、既存 surface の範囲で追加を検討する

## 検証

- 存在しない要件IDを検出できる
- 存在しない来歴IDを検出できる
- 存在しない受け入れ基準番号を検出できる
- 参照先IDから参照元ファイルを返せる
- `maintenance` では正本・中核層の変更に参照が無い場合、強めに警告または停止できる
- 初期状態では参照不足を助言に留められる
