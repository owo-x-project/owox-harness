# Phase8 経験を厚くする 要件

## 状態

実装済・検証済 (`docs/validation/20260617-Phase8-対話検証と粗の是正.md`)。`docs/concept/企画書.md` の 4.4 (判断と記憶を残す)・6.5 (継続的なプロジェクト改善)・要件 B群 (経験) を厚くする増分。
設計は `docs/decisions/20260616-Phase8-*.md`、進行は `docs/roadmap/20260612-完成ロードマップ.md` の Phase8。

## 背景

Phase7 までで「腐らず成長する」核 (腐敗検知・二層ルール・経験 IO) が立った。
本フェーズは、その上に「プロジェクトの記憶を厚くし、経験から技を育てる導線」を足す。

きっかけはマネージャ問いの4点 (`docs/handoff/20260616-Phase8開始-経験を厚くする.md` に経緯):

- 調査 (リサーチ) の成果に置き場が無い。来歴に紛れるか消える
- スキルのテストが「何を確かめるか」不明で作れない。空テストでも通る穴
- よく使う手順をスキルへ育てる導線が、AI の気づき任せで弱い
- canon 編集 tool (canon.add / canon.propose) の据わりの確認

owox-harness 自身と owlspec 開発に本ハーネスを使う前提なので、調査記憶と技の育成は実利用で効く。

## 目的

プロジェクトの調査記憶を検証つきで蓄積し、作業から技を育てる導線を強め、経験層を厚くする。
最小コンテキストとドメイン分離は崩さない。

## 作るもの

### 調査知識層

- `.owox/knowledge/` に1調査1ファイル。記録層 (decisions / tasks) を踏襲
- 必須属性: 調査日・ソース・要約。鮮度を経過日数で機械判定 (腐敗検知へ連動)
- 読みはオンデマンドのみ (注入しない)。tool で読み書き (canon 直読み禁止と整合)
- 更新は supersede 専用 (上書きせず新規で置換し旧を superseded)。来歴 (調査時点で何を知っていたか) を保つ
- 秘密走査を通す。ドメイン固有なので経験 export から除外

設計: `docs/decisions/20260616-Phase8-調査知識層.md`

### スキルテスト是正

- 契約 lint を全スキル共通の最低ゲートにする (owox が自動・人間が書かない)
- 「テストは依存する script / tool の振る舞いを確かめる」と定義を明確化
- 決定的にテストできる対象が無い純散文スキルは自動起動にできない方針を明確化
- 振る舞いの自然言語評価は Phase10 の opt-in・助言・ゲートにしない

設計: `docs/decisions/20260616-Phase8-スキルテスト是正.md`

### パターンからスキル育成の導線

- 使用履歴を追記専用で残す (tool / コマンド名と日付のみ・引数は残さない)
- 頻出する隣接手順を機械検出し、未スキル化なら助言する (腐敗検知と同じ流儀)。routine 検出と skill 化の要否は owox の行動軸で断定し提示。人間へ打診しない
- 育成は既存スキルライフサイクル (作成 → テスト合格 → 登録 → 昇格) へ流す。skill の登録・昇格 (正本化) は人間 gate。雛形自動生成はしない

軸の位置づけは `docs/decisions/20260627-判断2軸と対話kickoff.md`。

設計: `docs/decisions/20260616-Phase8-パターンからスキル育成.md`

### canon 編集の統一維持

- canon.add / canon.propose の統一を維持し、分割しない。説明文の簡潔化と不一致フィードバックで迷いを潰す

設計: `docs/decisions/20260616-canon編集の統一維持.md`

## 作らないもの

- 仕様層 (要件 対象外。仕様は当面コードで持つ。将来 owlspec)
- 振る舞いの自然言語評価をテストゲートにする (決定論の核を汚す。Phase10 opt-in へ)
- 使用パターンからのスキル雛形自動生成 (skill の正本化は人間 gate。owox が雛形を断定生成すると製品意図軸を侵す。`docs/decisions/20260627-判断2軸と対話kickoff.md`)
- canon 編集 tool の型別分割 (同一振る舞いの tool が並び逆に迷う)
- 使用履歴への引数記録 (秘密漏れ・肥大)

## 完了条件

- 調査記憶が tool で書け、オンデマンドで引け、古いものが腐敗検知で警告される
- 経験 export に調査記憶 (ドメイン固有) が混ざらない
- 全スキルが契約 lint を通らなければ登録されない。script 参照欠落・実行ビット欠落を弾く
- 頻出手順が助言で表面化し、そこからスキルライフサイクルへ進める
- canon.add / canon.propose が迷わず回る (実機 Codex)
- cargo / clippy 通過、MCP stdio・hook で機械確証、実機 Codex 対話で確認

## 検証

- 単体: 調査記憶の round-trip・supersede 遷移・鮮度閾値・秘密検出で failed・lookup 一致・export 除外、契約 lint (script 参照欠落 / 実行ビット欠落で draft)、使用履歴追記・頻出検出閾値・未スキル化 filter
- 結合: serve 経由の tool 呼び出し、next / verify.run の助言出力
- 手動 (実機 Codex): knowledge.add → lookup → get を直読みせず回す、頻出手順が提案される、canon.add / canon.propose の据わり
