# 差分文脈

## 状態

草案。次回以降に詳細を詰める。

## 背景

ブランチで変えた正本を、人間が merge 前に確認しやすくしたい。
ただし正本だけを見ると、なぜ変えたかが見えにくい。変更されたソースファイル一覧も同時に見せる。

tool は増やさず、既存の `context` / `review.lenses` / `verify.run` へ寄せる。

## 目的

- ブランチの変更地図を軽く見られる
- 正本変更と実装変更の関係を確認できる
- 正本差分レビューの入口になる
- 参照ID検査・庭師機能・腐敗検知へ再利用できる

## 対象

- ブランチ差分
- 作業ツリー差分
- 未追跡ファイル
- 正本面
- ソース・テスト・設定・生成物のファイル一覧

## 対象外

- ファイル本文の返却
- 生 diff の返却
- 新規 diff 専用 tool
- 自動レビュー結果の保存

## 仕様

`context` に scope を追加する。

```text
scope = "diff"
```

`diff` は正本差分専用ではなく、このブランチの変更地図。

返すもの:

- `base`: 比較元
- `changed_files`: 変更ファイル一覧
  - path
  - status: added / modified / deleted / renamed
  - kind: canon / docs / source / test / config / generated / unknown
- `canon_changes`: 正本面だけの抜粋
- `reference_summary`: 差分内の `owox:req:` / `owox:dec:` と壊れた参照数
- `review_hints`: 人間確認が必要そうな点

ファイル本文と生 diff は返さない。必要な時だけ通常の読み取りへ進む。

## 実現方針

- `context scope="diff"` が変更地図を返す
- `review.lenses` は正本面の変更がある時、既存の requirement / plan-alignment / design を強める
- `verify.run` は参照ID、未承認 gate、正本変更の来歴不足、生成物直接編集を確認する
- 正本差分レビューは review skill の流れで扱う

owox が提示する検証フロー（行動軸。owox が断定して示す。`docs/decisions/20260627-判断2軸と対話kickoff.md`）:

1. `review.lenses`
2. `verify.run`
3. `context scope="diff"`
4. 正本変更を先に確認
5. その後コード差分を確認

## 検証

- 変更ファイル一覧を状態つきで返せる
- 正本面だけ抜粋できる
- ソース・テスト・設定・生成物を分類できる
- ファイル本文と生 diff を返さない
- 正本変更ありの時にレビュー観点が強まる
- `verify.run` に参照ID検査結果を載せられる
