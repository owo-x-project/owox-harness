# 正本索引方針

## 状態

草案。初期実装の結論だけ固める。

## 背景

`target harness` 構成物の場所を機械的にたどる索引案はある。
ただし、専用 manifest を増やすと更新漏れ源になる。
今ある `context scope="codebase"` で読む場所の地図は返せる。

## 結論

初期実装では専用 manifest を作らない。
`context scope="codebase"` を正本探索の入口にする。

## 理由

- 正本を増やさずに済む
- repo から再計算できる情報だけで足りる
- `context` なら必要時だけ返せる
- manifest 更新漏れより codebase index 再計算の方が安全

## 対象

- `context scope="codebase"`
- codebase index cache
- docs/requirements

## 対象外

- `context scope="manifest"`
- `.owox-harness/manifest.json`
- manifest 専用 tool

## 検証

- 構成探索は `context scope="codebase"` で始められる
- 専用 manifest を前提にした実装が無い
- docs/requirements 間で矛盾しない
