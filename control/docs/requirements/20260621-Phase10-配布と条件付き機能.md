# Phase10 配布と条件付き機能

## 背景

Phase10 D群 (`docs/requirements/20260611-core-要件.md` の D群)。コア・規模・移植まで揃った製品を、他者が導入できる形にする。配布方式は技術スタックで既定 (`docs/decisions/20260611-技術スタック.md`)。設計は `docs/decisions/20260621-Phase10-配布とrelease正本.md`。

本要件は D群のうち「配布」と「release.toml」を扱う。eval (opt-in) とチーム権限は別スライス。

## 目的

owox-harness を GitHub Releases から checksum 検証つきで導入できるようにする。配布運用がある対象プロジェクトが配布方針・版・成果物検証を正本化できるようにする。

## 対象

- owox-harness 自身の配布: release workflow・setup.sh・install.ps1・SHA256SUMS・版
- 対象プロジェクト向け release.toml: 型・読込・検証・release.check MCP tool
- 人間向け導入文書 (README)

## 対象外

- eval (opt-in。別スライス)
- チーム権限 (役割・承認委譲。別スライス)
- 配布物への署名 (checksum で当面足りる。見直し条件で扱う)
- owox 内での hash 計算 (検査は対象プロジェクトへ委譲)
- macOS x86_64・Windows arm

## 成功条件

- 4種 (linux x86_64=`musl` / linux aarch64=`gnu` / macOS aarch64 / windows x86_64) の owox 実行ファイルが main の `.owox-version` 変更で組まれ Release へ添付される
- setup.sh / install.ps1 が成果物と SHA256SUMS を取得し checksum 照合してから配置する
- 照合失敗時に配置せず止まる
- `owox --version` で導入物の版が分かる
- `.owox/release.toml` が任意正本として読め、型検証され、無くても生成が通る
- release.check が版を取り出し、成果物存在を確認し、委譲検査の結果を封筒で返す
- 再生成で配布関連の生成物が壊れない

## 仕様

### 配布 workflow

- 配置: モノレポ root `.github/workflows/release.yml`。trigger は main の `control/.owox-version` 変更
- `.owox-version` は `owox-v<version>` 形式。Release tag とビルド時の `owox --version` はこの値へ追従する
- ビルド4種を native runner で。成果物は `owox-<target>.tar.gz` / `.zip`
- linux x86_64 は `x86_64-unknown-linux-musl` を使い、古い `glibc` 環境でも動くようにする
- 全成果物の SHA256 を `SHA256SUMS` へまとめ、`gh release create` で実行ファイルと共に添付

### setup.sh / install.ps1

- OS / CPU 判定 → 対応成果物名へ写像
- 版は引数 / 環境変数で指定、無指定は最新 `owox-v*` を解決
- 成果物 + SHA256SUMS を取得 → checksum 照合 → 照合 OK のみ配置
- 導入先既定 `~/.local/bin` (unix)・環境変数で変更可。Windows も既定先 + PATH 案内

### 版

- 通常ビルドの workspace 共通 version は `0.1.0` のまま。配布ビルドでは `.owox-version` から workflow が workspace version を注入する。owox に `--version` / `-V`

### release.toml

- `policy`: 文字列配列。案内のみ
- `[version]`: `file` + `pattern` (捕捉群1つの正規表現・行頭は複数行モード `(?m)`。読込時に正規表現の妥当性を検証)
- `[[artifacts]]`: `name` (期待成果物名)
- `[release] checks`: 委譲検査コマンド (verify.checks と同方式)
- 必須項目欠落・未知キー・不正な正規表現は型検証エラー

### release.check MCP tool

- 引数: dist ディレクトリ (既定はカレント)
- release.toml 読込 → 版抽出 → 成果物存在確認 → 委譲検査実行 → 封筒
- release.toml 不在は needs_human ではなく「配布運用なし」を案内

## 検証

- cargo test / clippy / fmt
- 型検証: release.toml の必須欠落・未知キー・不正正規表現を弾く
- setup.sh の shellcheck
- workflow yaml の構文確認
- 実機 (人間関与・未): main の `.owox-version` 変更 merge での workflow 実走・別環境での setup.sh 導入・release.check の dist 照合

## 完了条件
