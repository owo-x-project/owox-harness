# Repomix

## 調査日

2026-06-12

## 分類

文脈管理、repo 圧縮。

repo 内容を AI 向けにまとめる。

## 概要

Repomix は、repo を AI が読みやすい 1 つのファイルへまとめる道具。

token 数、秘密情報検査、圧縮、遠隔 repo 入力、MCP 周辺連携を持つ。
owox-harness の文脈最小化と安全な文脈出力で参考になる。

## 何がすごいか

- repo を AI 向け入力へ変換する体験が明確
- token 数を見える化する
- 秘密情報検査を含む
- 遠隔 repo も扱える
- 出力形式と除外設定が整っている

## owox-harness が取り入れる点

- target harness の文脈出力道具候補
- token 予算表示
- 秘密情報検査
- 除外規則
- 再現可能な文脈出力

## owox-harness が超える点

### task 単位

Repomix は repo 全体梱包が強い。
owox-harness は task に必要な文脈だけを選ぶ。

### 正本連動

Repomix は文脈出力道具。
owox-harness は正本、task、検証条件に文脈を結ぶ。

### 来歴

Repomix は出力を作る。
owox-harness は何を含め、何を除いたかを来歴に残す。

### 安全境界

Repomix は秘密情報検査を持つ。
owox-harness は人間ゲートと合わせて、安全に渡せる文脈だけを使う。

## owox-harness が負けている点

- 文脈梱包の完成度
- token 数表示
- 秘密情報検査
- 出力形式
- CLI / Web の導入体験

## 勝つための判断

Repomix は「AI に repo を渡す」で強い。
owox-harness は「なぜ、どの範囲を、どの task のために渡すか」で勝つ。

梱包機能は自作前に Repomix 連携を検討する。

## 見直し条件

- Repomix が task 単位文脈を持った時
- Repomix が正本や検証条件と結びついた時
- Repomix が人間ゲートを持った時
- Repomix が target harness 生成へ寄った時
