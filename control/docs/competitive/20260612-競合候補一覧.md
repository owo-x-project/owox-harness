# 競合候補一覧

## 状態

調査継続。

## 範囲

調査日: 2026-06-12。

「同じ分野」は次を含める。

- target harness 相当を生成、配置、運用するもの
- 仕様、計画、タスク、検証、文脈を AI 開発の正本にするもの
- 複数 AI CLI や複数実行エージェントを束ねるもの
- 実行エージェントそのもの
- 文脈、記憶、repo 圧縮を支えるもの

完全網羅は断定しない。
公開一次情報で確認できた候補と、名前のみ確認した候補を分ける。

## 除外または低優先

- Claude Code
  - 理由: OSS ではない。ただし市場基準として観測する

- Cursor
  - 理由: OSS ではない。ただし利用者期待値として観測する

- GitHub Copilot 実行エージェント
  - 理由: OSS ではない。ただし GitHub 上の agent 管理体験として観測する

- Devin
  - 理由: OSS ではない。ただし非同期 agent の市場基準として観測する

- Jules
  - 理由: OSS ではない。ただし GitHub Agent HQ 周辺として観測する

- Bytebot
  - 理由: 自己ホスト型デスクトップ agent。コード変更特化ではない

- 汎用 agent 基盤
  - 例: LangGraph、AutoGen、CrewAI
  - 理由: owox-harness の直接比較ではない。Deep Agents のように実行基盤 / コード変更へ寄る時だけ個別候補へ上げる
