# Open SWE

## 調査日

2026-06-12

## 分類

実行エージェント管理、agent 実行基盤。

組織内の非同期 coding agent を作るための OSS。

## 概要

Open SWE は、組織内で使う coding agent を作るための実行基盤。

隔離環境、Slack / Linear 起動、subagent、PR 作成、AGENTS.md 取り込みを持つ。
owox-harness の target harness が制御する下位層、または統合先になりうる。

## 何がすごいか

- 非同期実行エージェントの現実的な構成を持つ
- task ごとに隔離環境を使う
- Slack / Linear / PR という業務導線に近い
- AGENTS.md を文脈として読む
- agent harness を既存基盤へ組み合わせる考え方

## owox-harness が取り入れる点

- task ごとの隔離環境
- 外部サービス起点の作業開始
- AGENTS.md 取り込み
- subagent と実行環境の分離
- PR 作成までの非同期流れ

## owox-harness が超える点

### 正本生成

Open SWE は実行する基盤が強い。
owox-harness は実行前に target harness 正本を生成、検証、維持する。

### 人間判断

Open SWE は隔離と権限境界で安全性を作る。
owox-harness は判断点を正本化し、承認の来歴を残す。

### 汎用性

Open SWE は組織内 agent 実装に近い。
owox-harness は複数 AI CLI と target harness 形式に広げる。

## owox-harness が負けている点

- 実行環境
- 非同期 PR 作成
- Slack / Linear 連携
- 隔離環境の具体性
- 実装基盤としての完成度

## 勝つための判断

Open SWE と実行基盤で競わない。

owox-harness は、Open SWE のような実行基盤に渡す前の正本、制約、検証、判断を持つ。
「実行 agent の上に立つ制御層」として位置づける。

## 見直し条件

- Open SWE が target harness 生成を持った時
- Open SWE が task 正本と検証 gate を持った時
- Open SWE が MCP resource / tool で正本操作を標準化した時
- owox-harness が非同期 agent 実行を自前で持ちたくなった時
