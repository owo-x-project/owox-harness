# Beads

## 調査日

2026-06-12

## 分類

タスク管理。

## 状態

未確認調査票。

公開一次情報は未確認。
ただし owox-harness のタスク管理判断で、超える対象として既に扱っている。

## 概要

Beads は、AI 作業における task 管理の比較対象。

現時点の手元文書では、腐る task 管理の代表例として扱う。
特に、自己申告 done、判断との結びつきの薄さ、古い task の放置が比較軸。

## 何がすごいか

一次情報未確認のため断定しない。

想定上の強み:

- 軽い task 管理
- ready の概念
- AI が次にやることを見失わない構造
- 既存運用へ入れやすい可能性

## owox-harness が取り入れる点

- task を AI 作業の背骨にする
- ready を扱う
- task の作成、依存、完了を明示する
- 既存 task からの移行入口を検討する

## owox-harness が超える点

### done 検証

owox-harness は done を自己申告にしない。

### 来歴連動

task の作成、完了、破棄の理由を来歴へ残す。

### 腐敗検知

放置、孤立、重複、永久停止、未検証 done を検出する。

### 人間ゲート

範囲外、不可逆、外部送信、正本昇格を task 上で止める。

## owox-harness が負けている点

一次情報未確認のため断定しない。

想定上の負け:

- 導入の軽さ
- 既存利用者
- task 操作の手軽さ
- Beads から離れたくない利用者への移行抵抗

## 勝つための判断

Beads を「置き換える task 管理」として見る。

owox-harness の勝ち筋は、task を便利に持つことではなく、腐らせないこと。

## 見直し条件

- 公式 URL を確認した時
- Beads の正確な task model を確認した時
- Beads が検証 gate、来歴、腐敗検知を持つと分かった時
- Beads import を実装する時
