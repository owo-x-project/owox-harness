# Mem0

## 調査日

2026-06-12

## 分類

記憶管理。

agent 向け長期記憶を扱う。

## 概要

Mem0 は、AI agent とアプリ向けの記憶基盤。

利用者ごとの記憶、長期記憶、SDK、MCP 連携を前面に出す。
owox-harness から見ると、経験保存と文脈再利用で比較対象になる。

## 公開値

2026-06-12 時点:

- GitHub 星数: 約 58.4k
- 分岐数: 約 7.3k
- 履歴数: 1,708
- SDK、MCP 連携あり
- OSS と cloud 版あり

## 何がすごいか

- 記憶管理領域で認知が大きい
- agent 向け長期記憶を分かりやすく売っている
- SDK と MCP 連携がある
- 利用者ごとの記憶を扱う
- 導入先が広い

## owox-harness が取り入れる点

- 長期記憶の利用者体験
- MCP 経由の記憶操作
- SDK と道具の両方を置く構成
- 記憶の追加、検索、削除の基本操作
- 記憶を導入価値として説明する方法

## owox-harness が超える点

### 記憶の責務

Mem0 は記憶基盤。
owox-harness は記憶を正本、task、検証、人間判断へ結ぶ。

### 混入防止

Mem0 は個別化が強い。
owox-harness はドメイン固有情報を汎用経験へ混ぜない。

### 来歴

Mem0 は記憶を取り出す。
owox-harness はどの記憶を判断に使ったか残す。

### 移植

Mem0 は記憶基盤。
owox-harness は経験を export / import し、target harness へ移す。

## owox-harness が負けている点

- 公開利用者
- 記憶基盤の完成度
- SDK
- MCP 連携
- 記憶を価値として伝える力

## 勝つための判断

Mem0 は「記憶を持つ agent」で強い。
owox-harness は「記憶を使ってよい場面、使った理由、混ぜてはいけない境界」で勝つ。

記憶基盤は自作前に Mem0 連携を検討する。

## 見直し条件

- Mem0 が開発 task 正本へ寄った時
- Mem0 が検証や人間ゲートを持った時
- Mem0 が target harness 生成へ寄った時
- Mem0 が経験 export / import と重なった時
